# Checklist de préparation déploiement monitoring

## Connectivité
- [ ] IP / ports / endpoints validés
- [ ] protocole choisi par équipement
- [ ] droits d'accès confirmés
- [ ] test de communication réalisé

## Données
- [ ] registre des points créé
- [ ] unités et scaling validés
- [ ] timestamps cohérents
- [ ] criticité des points définie

## Historisation
- [ ] choix InfluxDB ou TimescaleDB fixé
- [ ] politique de rétention définie
- [ ] stratégie store & forward définie
- [ ] stratégie UTC confirmée

## Dashboards
- [ ] vue maintenance définie
- [ ] vue exploitation définie
- [ ] vue robot définie si nécessaire
- [ ] vue OEE / énergie définie si nécessaire

## Alertes
- [ ] matrice d'alertes validée
- [ ] niveaux de sévérité définis
- [ ] destinataires définis
- [ ] temporisations définies

## Exploitation
- [ ] propriétaire technique identifié
- [ ] propriétaire métier identifié
- [ ] procédure de support définie
- [ ] critères d'acceptation définis
