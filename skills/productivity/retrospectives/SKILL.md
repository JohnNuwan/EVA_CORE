# Rétrospectives — Référence Complète

## Contexte & Déclencheur
Utiliser quand l'utilisateur demande : rétrospective, retro, amélioration continue, kaizen, Sprint Retrospective, facilitation, action items, prime directive.

---

## 1. Fondamentaux

### Prime Directive (Norman Kerth)
> « Indépendamment de ce que nous découvrons, nous comprenons et croyons vraiment que chacun a fait de son mieux, compte tenu de ce qu'il savait à ce moment-là, ses compétences et ses capacités, les ressources disponibles et la situation dans laquelle il se trouvait. »

### 5 Phases d'une Rétro (Scrum Guide + Esther Derby)

| Phase | Durée | Objectif |
|-------|-------|----------|
| **1. Set the Stage** | 5 min | Créer un espace sûr, rappeler la Prime Directive |
| **2. Gather Data** | 10-15 min | Collecter les faits, les émotions, les événements |
| **3. Generate Insights** | 15-20 min | Analyser, trouver les causes racines |
| **4. Decide What to Do** | 10-15 min | Actions concrètes, prioritaires (1-3 max) |
| **5. Close the Retro** | 5 min | Vérifier satisfaction, remercier |

## 2. Formats de Rétro

### 2.1 Start / Stop / Continue (le plus commun)

```
🟢 Start (Ce qu'on commence)
- ...
🟡 Stop (Ce qu'on arrête)
- ...
🔵 Continue (Ce qu'on garde)
- ...
```

**Quand :** Équipe stable, rythme établi, besoin d'ajustements simples.

### 2.2 Sailboat (Bateau à Voile)

```
🌬️ Vents porteurs (ce qui nous aide à avancer)
- ...
⚓ Ancres (ce qui nous ralentit)
- ...
🏝️ Île (notre objectif, vision)
- ...
⛰️ Rochers (risques à éviter)
- ...
```

**Quand :** Équipe qui a besoin de vision, motivation, ou après un échec.

### 2.3 4L (Liked / Learned / Lacked / Longed For)

```
❤️ Liked (Ce qu'on a aimé)
- ...
📚 Learned (Ce qu'on a appris)
- ...
💔 Lacked (Ce qui a manqué)
- ...
🌟 Longed For (Ce qu'on aurait souhaité)
- ...
```

**Quand :** Équipe introspective, besoin de profondeur émotionnelle.

### 2.4 Mad / Sad / Glad

```
😡 Mad (Ce qui nous a fâchés)
- ...
😢 Sad (Ce qui nous a attristés)
- ...
😊 Glad (Ce qui nous a rendus heureux)
- ...
```

**Quand :** Équipe qui a besoin d'exprimer des émotions, rétro après un sprint difficile.

### 2.5 Timeline (Chronologie)

```
Sprint J1 ────────────────────── Sprint J14
    │               │                  │
    ├─ Victoire 1  ├─ Problème 1     ├─ Victoire 3
    ├─ Problème 2  ├─ Victoire 2     ├─ Apprentissage
```

**Quand :** Sprint long, beaucoup d'événements, besoin de vue d'ensemble.

### 2.6 Speed Boat (Moteur Hors-Bord)

```
🚀 Moteur (ce qui nous propulse)
- ...
🪝 Ancres (ce qui nous retient)
- ...
```

**Variante :** Les ancres sont des problèmes identifiés, on vote pour les remonter.

### 2.7 Drop / Add / Keep / Improve (DAKI)

```
🗑️ Drop (Abandonner)
- ...
➕ Add (Ajouter)
- ...
📦 Keep (Garder)
- ...
📈 Improve (Améliorer)
- ...
```

**Quand :** Équipe qui change de workflow ou de stack.

### 2.8 Fun / Done / Learn (Mountain Goat)

```
🎉 Fun (Moments agréables)
- ...
✅ Done (Complété)
- ...
📚 Learn (Apprentissages)
- ...
```

**Quand :** Équipe qui a besoin de légèreté, ou rétro informelle.

## 3. Techniques de Facilitation

### 3.1 Icebreakers (5 min)

| Technique | Description |
|-----------|-------------|
| **Check-in météo** | ☀️/⛅/🌧️ — comment je me sens |
| **One Word** | Un mot pour décrire le sprint |
| **Roses & Épines** | Un bon moment, un mauvais moment |
| **Energy Level** | 1-5 sur l'énergie personnelle |
| **Appreciation** | Remercier quelqu'un |

### 3.2 Vote et Priorisation

| Technique | Usage |
|-----------|-------|
| **Dot Voting** | Chacun 3 dots, priorise les sujets |
| **Fist of Five** | 0-5 accord sur une action (0=bloquant, 5=total accord) |
| **Thumbs** | 👍/👎/🫤 pour avis rapide |
| **100 Points** | Distribuer 100 points entre les options |

### 3.3 Root Cause Analysis

**5 Why (5 Pourquoi)** — voir skill risk-management
**Fishbone** — voir skill risk-management

### 3.4 Silent Brainstorming

```
1. Chacun écrit individuellement (5 min) — pas de discussion
2. On partage à tour de rôle — pas de débat
3. On regroupe par thème
4. On vote pour les sujets importants
5. On discute les priorités
```

**Avantage :** Pas de domination des voix fortes, introvertis inclus.

## 4. Actions Concrètes — SMART

### 4.1 Transformer les Insights en Actions

```
❌ « Améliorer la communication »
✅ « Ajouter une synchronisation de 15min entre devs et QA chaque matin à 10h, pendant 2 semaines. Responsable : Alice. »
```

### 4.2 SMART Criteria
```
S — Spécifique (quoi, qui, où)
M — Mesurable (comment savoir si c'est fait)
A — Atteignable (réaliste)
R — Relevant (pertinent pour le problème)
T — Time-bound (date limite)
```

### 4.3 Règle des 1-3 Actions Max
- Ne pas prendre plus de 3 actions par rétro
- 1 action = 1 responsable
- Deadline claire (avant la prochaine rétro)
- Review à la rétro suivante

## 7. Rétros Spéciales

### 7.1 Projet Kick-off Retro
```
Avant même de commencer : 
- Quels risques anticipons-nous ?
- Qu'est-ce qui a marché dans les projets précédents ?
- Quels sont nos espoirs et craintes ?
```

### 7.2 Release Retro
```
- Qu'avons-nous appris sur le produit ?
- Les hypothèses étaient-elles correctes ?
- Quels métriques montrent du succès ?
- Qu'est-ce qu'on aurait aimé savoir avant ?
```

### 7.3 Pre-mortem (Klein, 2007)
```
Imaginez que le projet a échoué, c'est sûr. 
Pourquoi ? → Identifier les risques avant qu'ils arrivent.
```

### 7.4 Post-mortem (Incident Analysis)
```
1. Chronologie des événements
2. Détection, réponse, résolution
3. Causes racines
4. Actions correctives (mitigation)
5. Leçons apprises
```

## 8. Rétro Anti-Patterns

| Anti-Pattern | Symptôme | Correction |
|-------------|----------|-----------|
| **Complainfest** | Tout le monde se plaint, rien ne change | Focus sur solutions, action items |
| **No Action Ever** | Les mêmes sujets reviennent chaque sprint | 1 action non faite = priorité absolue |
| **PO ou Manager présent** | L'équipe n'ose pas parler franchement | Rétro sans PO/manager |
| **Too Long** | 3h de rétro pour 2 semaines | Timer strict, time-box |
| **Too Short** | 15 min, "tout va bien" | Approfondir, utiliser des formats |
| **Blame Game** | On cherche le coupable | Prime Directive, focus processus |
| **Retro Fatigue** | Équipe en a marre de "réfléchir" | Alterner les formats, garder 30 min |
| **Pas de priorité** | 50 actions, rien fait | 1-3 actions max, SMART |

## 9. Mesure d'Impact des Rétros

```python
def retro_effectiveness_score(actions_completed, satisfaction, n_sprints):
    """Score d'efficacité des rétros (0-100)."""
    action_rate = actions_completed / (n_sprints * 3)  # 3 actions max par sprint
    sat_score = sum(satisfaction) / len(satisfaction) * 20  # /5 → /100
    
    return round((action_rate * 50 + sat_score * 50), 1)

# Exemple
print(retro_effectiveness_score(8, [3.5, 4.0, 3.2, 4.5], 4))
# → 77.5 / 100
```
