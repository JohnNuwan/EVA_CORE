# OPC UA Information Modeling - points de gouvernance

A figer systématiquement :
- politique de namespaces
- stratégie de NodeIds stables
- conventions BrowseName / DisplayName
- séparation types / instances / méthodes / événements
- alignement Companion Specification ou modèle métier

Vérifications minimales :
1. client tiers capable d'interpréter le modèle
2. NodeIds non éphémères
3. références cohérentes
4. export NodeSet validable
