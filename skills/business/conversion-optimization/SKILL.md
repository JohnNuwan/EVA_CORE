# Optimisation de la Conversion (CRO)

Compétence de niveau expert en Conversion Rate Optimization — méthodes systématiques pour augmenter le pourcentage de visiteurs qui réalisent une action souhaitée.

## 1. Fondamentaux du CRO

### Définitions
- **Taux de conversion** : % de visiteurs qui accomplissent un objectif (achat, inscription, lead)
- **Micro-conversions** : Petites actions menant à l'objectif principal (ajout panier, consultation page, téléchargement)
- **Macro-conversions** : Objectif final (vente, inscription abonnement)
- **Baseline** : Taux de conversion actuel (mesure avant toute optimisation)
- **Lift** : Amélioration relative (ex: +20% de conversions)

### Processus CRO (Méthodologie)
1. **Research** : Données quantitatives (analytics) + qualitatives (heatmaps, surveys, user testing)
2. **Hypothesis** : "Si [changement], alors [résultat], parce que [insight]" — formaté clairement
3. **Priorisation** : ICE Score (Impact × Confidence × Ease) ou PXL Framework
4. **Test** : A/B test ou split test avec significativité statistique
5. **Analyse** : Résultat gagnant, perdant, non concluant — documenter les apprentissages
6. **Implémentation** : Déployer le gagnant, itérer les perdants
7. **Répéter** : Le CRO est un processus continu, pas un projet ponctuel

## 2. Recherche et Analyse

### Analyse Quantitative
- **Funnel Analysis** : Entonnoir de conversion, taux de passage entre chaque étape
- **Drop-off points** : Étapes avec le plus grand abandon
- **Segmentation** : Mobile vs desktop, nouveau vs retour, source de trafic
- **Cohort analysis** : Variation de conversion par cohorte
- **Attribution** : Conversion assistée, multi-touch

### Analyse Qualitative
- **Heatmaps** : Click maps, move maps, scroll maps (Hotjar, Crazy Egg, Clarity)
- **Session recordings** : Replay des sessions utilisateur, identifier les frictions
- **On-site surveys** : "Qu'est-ce qui vous a empêché d'acheter aujourd'hui ?"
- **User testing** : Tests utilisateurs (UsabilityHub, UserTesting, Lookback)
- **Customer interviews** : Entretiens qualitatifs, jobs to be done

### Audit d'Entonnoir
- **Par étape** : Chaque page/action de l'entonnoir est une occasion d'optimisation
- **Formulaires** : Nombre de champs, labels, validation, erreurs, mobile-friendly
- **Checkout** : Processus en une page vs multi-étapes, guest checkout
- **Pricing** : Présentation des prix, comparaison, ancrage, décoy pricing
- **Landing pages** : Cohérence message-source, vitesse, mobile

## 3. Psychologie de la Conversion

### Principes de Persuasion (Cialdini)
1. **Réciprocité** : Offrir un bonus, un guide gratuit, un échantillon
2. **Rareté** : Stock limité, offre limitée dans le temps
3. **Autorité** : Logos, certifications, badges de confiance
4. **Preuve sociale** : Avis, témoignages, nombre d'utilisateurs, médias
5. **Sympathie** : Visages souriants, similarité, personnalisation
6. **Cohérence** : Petits engagements progressifs

### Biais Cognitifs
- **Ancrage** : Premier prix vu = référence (afficher le prix le plus élevé en premier)
- **Aversion à la perte** : "N'oubliez pas votre panier" vs "Complétez votre achat"
- **Paradoxe du choix** : Trop d'options = paralysie (idéal : 3-4 options max)
- **Effet de dotation** : Valeur perçue plus élevée quand on possède déjà
- **Preuve sociale implicite** : "X personnes achètent ce produit en ce moment"
- **Scarcity** : Compteur de stock, temps restant
- **Effet Google** : Confiance instantanée dans ce qui est premier dans les résultats

### Heuristics (LIFT Model — WiderFunnel)
- **Value Proposition** : Clarté de la proposition de valeur
- **Relevance** : Adéquation message- audience
- **Clarity** : Clarté du message, pas de confusion
- **Distraction** : Éliminer les éléments qui distraient du CTA
- **Anxiety** : Réduire les inquiétudes (sécurité, garantie, politique de retour)
- **Urgency** : Créer une raison d'agir maintenant

## 4. Tests A/B et Expérimentation

### Design de Test
- **A/B Test** : Version A (contrôle) vs version B (variante) — un seul changement
- **Multivariate Test** : Plusieurs changements combinés (nécessite plus de trafic)
- **Split URL Test** : Deux URLs différentes (page entièrement différente)
- **MVT** : Test multivarié — chaque combinaison d'éléments est une variante

### Significativité Statistique
- **P-value** : < 0.05 pour confiance 95%
- **Power analysis** : Puissance ≥ 80% pour détecter un effet
- **Sample size minimum** : Calculateur (ex: Optimizely Sample Size Calculator)
- **Duration** : Minimum 7 jours (couvrir les cycles hebdomadaires)
- **Peeking** : Ne pas regarder les résultats avant la fin du test
- **Sequential testing** : Alternative au peeking (Google's Bayesian approach)

### Outils de Test
- **Google Optimize** : Disparu (mars 2024) — alternatives : AB Tasty, VWO, Optimizely
- **VWO** : Testing + heatmaps + surveys
- **Optimizely** : Enterprise, feature flags, personalization
- **AB Tasty** : Testing + personalization AI
- **Kameleoon** : AI-driven testing
- **Server-side testing** : Contrôle total, pas de flicker (Optimizely Full Stack)

## 5. Landing Pages et CTA

### Landing Page Best Practices
- **Hero section** : Headline claire, sous-headline, CTA au-dessus de la ligne de flottaison
- **Headline** : Unique Selling Proposition (USP) en moins de 8 mots
- **Social proof** : Logos clients, témoignages, avis, chiffres
- **Visual hierarchy** : Taille, contraste, espacement — guider l'œil vers le CTA
- **Form simplification** : Moins de champs = plus de conversions (règle : commencer par ce qui est nécessaire, ajouter des étapes après)
- **Mobile-first** : Design d'abord pour mobile, puis desktop
- **Loading speed** : < 3s LCP, < 100 KB page weight, images optimisées

### CTA (Call-to-Action)
- **Wording** : Action-oriented, bénéfice (ex: "Commencez mon essai gratuit", "Réservez ma démo")
- **Couleur** : Contraste élevé avec le fond, test A/B recommandé
- **Taille** : Suffisamment grand pour être cliquable sur mobile (48x48px min)
- **Position** : Au-dessus de la ligne de flottaison, + scroll (multiples dans la page)
- **Urgency** : "Offre limitée", "Plus que 3 places"
- **Forme** : Bouton plutôt que lien texte (meilleur taux de clic)

### Formulaire Conversion
- **Nombre de champs** : Moins = mieux. Chaque champ supplémentaire réduit le taux de conversion
- **Single column** : Plus lisible que multi-colonnes
- **Inline validation** : Feedback immédiat (pas après soumission)
- **Progress indicator** : Pour formulaires multi-étapes
- **Guest checkout** : Option sans création de compte (augmente taux de 20-40%)
- **Trust badges** : SSL, paiement sécurisé, garantie satisfait ou remboursé

## 6. Mobile et Cross-device

### Mobile Conversion
- **Thumb zone** : Actions importantes dans la zone accessible au pouce (bas de l'écran)
- **Form optimization** : Labels visibles, champs larges, clavier adapté (type=tel, email)
- **Touch targets** : ≥ 48x48px, espacement minimal 8px
- **Page speed** : Core Web Vitals mobiles prioritaires
- **Responsive CTA** : Bouton "sticky" en bas de l'écran sur mobile

### Cross-device
- **Panier persistant** : Connecter les appareils (compte utilisateur)
- **Email recovery** : Abandon panier traité cross-device
- **User ID tracking** : GA4 avec User-ID pour parcours cross-appareil
- **Synchronisation** : Continuer sur mobile ce qui a été commencé sur desktop

## 7. Personalization

- **Geo-targeting** : Contenu, prix, devise adaptés à la localisation
- **Traffic source** : Landing pages différentes selon l'origine (Google Ads, Facebook, Email)
- **Behavioral** : Recommandations basées sur l'historique de navigation
- **Returning visitor** : Message différent pour les visiteurs connus
- **Device-based** : Expérience mobile différente du desktop
- **Segmentation avancée** : Personnalisation par secteur, taille entreprise, rôle

## 8. KPIs et Reporting

### KPIs Essentiels
- **Taux de conversion** : Macro et micro-conversions
- **Taux de rebond** : Qualité du trafic (rebond < 40% = bon)
- **Taux d'abandon panier** : Checkout abandonment (moyenne ~70%)
- **Taux de complétion formulaire** : % de formulaires soumis
- **Revenue Per Visitor (RPV)** : Revenu total / visiteurs
- **Average Order Value (AOV)** : Panier moyen
- **Customer Acquisition Cost (CAC)** : Coût total d'acquisition

### Reporting
- **Test results** : Lift, confidence interval, sample size, duration
- **Incremental impact** : Conversions additionnelles * valeur moyenne
- **Learnings log** : Base de connaissances des tests (ce qui marche, ce qui ne marche pas)
- **Executive summary** : Synthèse mensuelle des optimisations et résultats

## 9. Pièges et Antipatterns

- **Tester trop de choses à la fois** : Impossible d'isoler la cause du changement
- **Arrêter un test trop tôt** : Dès que p < 0.05, mais pas de validation temporelle
- **Ignorer la segmentation** : Résultat gagnant dans un segment, perdant dans un autre (Simpson's paradox)
- **Changer le test en cours** : Ajouter une variante après le début du test
- **Sur-optimiser une micro-conversion au détriment de la macro** : Exemple : trop de pop-ups = +inscriptions mais -achats
- **Oublier le mobile** : Test A/B uniquement desktop
- **Dark patterns** : Tactiques trompeuses (opt-out difficile, frais cachés) → mauvaise réputation à long terme
- **CRO sans trafic** : Pas assez de visiteurs pour atteindre la significativité → tester sur les pages les plus visitées en priorité
